# 程序员学原始递归函数

> 原文: https://matklad.github.io/2024/08/01/primitive-recursive-functions.html

其实这篇谈的是关于配置语言的问题，许多人在讨论为什么用配置语言代替通用编程语言时会说因为它们**不是图灵完备的**，所以可以避开停机问题这种复杂的问题，但这样的描述并不好。*原始递归函数(Primitive Recursive Functions)*不是图灵完备的，但是它可以轻松写出复杂度奇高的运算，在几百万年后可以停机和不停机对人类来讲没什么区别。

按照matklad的设想，一个好的配置语言应该：

+ *确定(deterministic)* - 他以python的`id([])`为反例，这个表达式每次得到的结果都不一样！配置语言经常跟某种缓存和渐进运算机制一起使用，不确定性会毁了这一切。】
+ *良定义(well-defined)* - matklad又用python举例子，如果在编译Python时关掉ASLR，并且使用特定的内存分配器，那么`id([])`可以得到确定的结果，但这结果没法预测，不同实现之间行为也不保证一样。
+ *纯(pure)* - 有副作用缓存机制不好办。
+ *安全与沙盒机制(security and sandboxing)* - 不能让一个恶意配置文件烧坏你的CPU。
+ *简单(simple)* - 最好写不出复杂度会爆的递归。
